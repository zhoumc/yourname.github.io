First try to write something serious

ÂèëË°®‰∫é 2016-10-19 |

又是一篇水文，嗯（认真脸）。
不过话说回来，今天确实解决了几个问题耶，那就记录一下吧，以后养成好习惯。之后再讲几句废话。格式什么的以后再仿照大神学呗，一点点来。
具体内容如下：

Problem Solved 1

导师要求比赛啦，完全木有思路，但还好有大神队友，一周的时间就从完全无从下手到大致有思路了，棒！分成了几个部分走，不过它的数据还是有点问题。

我们的题目的难点就在于他给的字段非常多，数据量也很大，但是最后也就是一个’二分类’的问题，所以前期的数据处理与特征提取就显得尤为重要。

在第一步我们就面临着一个把数据导入的问题。首先就是考虑导入数据库，tsv文件用excel可以打开不过要分两趟，因为几百万条的数据量确实很大了，用记事本打开也是只能看一眼，编程难度会很大。如果能够直接导入数据库的话，写SQL语句会比较简单。tsv文件用Mac上的MySQl workbranch打不开，SQL server打开中文字段乱码，泽清说可以写个小程序。今早我也去下了个SQL server，用它自带的导入工具确实可以导入几个表，但一有中文字段就报错。错误如下：

错误 0xc02020a1: 数据流任务 1: 数据转换失败。列“OVERSEE_RESON”的数据转换返回状态值 4 和状态文本“文本被截断，或者一个或多个字符在目标代码页中没有匹配项。”。
(SQL Server 导入和导出向导)

错误 0xc020902a: 数据流任务 1: “源 - s_info_oversee_tsv.输出[平面文件源输出].列[OVERSEE_RESON]”由于发生截断而失败，而且针对“源 - s_info_oversee_tsv.输出[平面文件源输出].列[OVERSEE_RESON]”的截断行处理设置指定截断时出错。在指定组件的指定对象上出现截断错误。
(SQL Server 导入和导出向导)

错误 0xc0202092: 数据流任务 1: 处理文件“G:\BaiduYunDownload\客户画像\客户画像\s_info_oversee.tsv”的数据行 2 时出错。
(SQL Server 导入和导出向导)

错误 0xc0047038: 数据流任务 1: SSIS 错误代码 DTS_E_PRIMEOUTPUTFAILED。源 - s_info_oversee_tsv 的 PrimeOutput 方法返回了错误代码 0xC0202092。管道引擎调用 PrimeOutput() 时该组件返回了一个失败代码。虽然该失败代码只与该组件有关，但这个错误是致命的，管道已停止执行。可能在此之前已经发出错误消息，提供了有关失败的详细信息。
(SQL Server 导入和导出向导)

解决方式：

将中文字段类型改成unicode 字符串。

将长度改大。

Problem Solved 2

双屏的问题。恩。激动的还发了个票圈嘚瑟一下。主要就是靠一个DVI转VGA转换器。(此处感谢超杰大哥~)
当时确实看了一下显卡是集显，两个VGA口也都是好的，就是连一个另一个找不到，这个确实也不清楚具体的原因。配置的话就是:

右键->显示->检测，扩展显示器。TA-DA!!

激动的又去找壁纸，为了以后能保持新鲜感，那么就经常登Wallpaper,定期换壁纸吧。

本文重点-废话来啦

明天又要开组合了蓝瘦，啥也没做怎么汇报，害怕，但是估计就能水过去，水过去就到了最近最大的一件事了->跑马！，一个人去陌生的地方，感觉好不一样呀。

今天还学会了游泳！，只去了五次噢，nice！而且健身没偷懒都上了大重量呢。读研期间两个的小目标（游泳hi跑半马）竟然这么快就实现了。

所以，得出什么结论呢，（前方鸡汤）：任何东西就是时间问题，对未来没什么好担忧的，只要一步一个脚印，迷茫着迷茫着也发现自己走了那么远。

结语

我其实是一个纸媒至上者，明明有电子书呀，也可以写博客，但是就是还是喜欢看纸质的，然后写一遍也能够理解，不涂不画真的就好难受，（矫情！）。但是坚持吧，电子的东西看上去更有逻辑一些（主要是能装x)，而且也不大可能会丢吧。

快捷键每天再学点。

小敏程，GO！渣渣不可能永远是渣渣，努力学没有学不会的！
